### 1.3 온보딩 경험 설계

#### 원칙: "쓰면서 배운다" (Learn by Writing)

온보딩은 별도의 튜토리얼 화면이 아니라, 실제 에디터 안에서 글을 쓰는 행위를 통해 자연스럽게 진행된다.

#### 온보딩 시퀀스 (첫 방문 시)

```
Step 1: Welcome Screen (전체 화면, 3초 자동 전환 또는 클릭)
  ┌─────────────────────────────────────────────┐
  │                                             │
  │          [Pen Line 아이콘]                   │
  │                                             │
  │       "Write. Prove. Trust."                │
  │                                             │
  │   세 단어로 HumanWrites의 본질을 전달       │
  │                                             │
  │          [Start Writing]                    │
  │                                             │
  └─────────────────────────────────────────────┘

Step 2: Ghost Text Guide (에디터 안)
  - 에디터에 연한 회색 가이드 텍스트가 미리 표시
  - "여기에 제목을 입력하세요..." (제목 영역)
  - "글을 쓰기 시작하면, HumanWrites가 조용히 당신의 글쓰기를
     지켜봅니다. 타이핑 리듬을 분석하고, 글이 완성되면
     '사람이 직접 썼다'는 인증서를 발급합니다."
  - 사용자가 타이핑을 시작하면 가이드 텍스트는 자연스럽게 사라짐

Step 3: Contextual Tooltips (글쓰기 중 순차 등장)
  - 첫 100단어 작성 후:
    "Cmd+I로 통계와 리뷰를 볼 수 있습니다" (에디터 우측 하단, 5초 후 사라짐)
  - 첫 맞춤법 피드백 발생 시:
    인라인 밑줄 위에 작은 "AI가 발견한 제안입니다. 호버하여 확인하세요" 툴팁
  - 첫 문서 완성(500단어 이상) 시:
    "글이 인증 준비가 되었습니다. Publish로 인증서를 발급하세요" 알림

Step 4: 온보딩 완료
  - 3가지 컨텍스트 팁을 모두 본 후 온보딩 상태 완료 마킹
  - 이후 방문에서는 팁 표시 안 함
  - 설정에서 "가이드 다시 보기" 옵션 제공
```

#### 온보딩 스킵 처리

- 모든 가이드 텍스트/툴팁에 "건너뛰기" 옵션
- 설정 > 일반 에서 "온보딩 초기화" 가능
- 기술 숙련 사용자는 즉시 글쓰기 시작 가능

### 1.4 글쓰기 집중 + 인증/AI 통합 UX 전략

#### 핵심 원칙: "존재하되 사라져라" (Be Present, Be Invisible)

```
글쓰기 집중도 스펙트럼:

[최대 집중] ◀──────────────────────────▶ [최대 보조]

Focus Mode     Ambient Mode     Review Mode     Settings
  |                |                |              |
  |  피드백 0%     |  피드백 30%    |  피드백 100% |  설정 100%
  |  UI 0%         |  UI 5%        |  UI 40%      |  UI 100%
  |  인증 숨김     |  인증 수동     |  인증 표시   |  인증 관리
```

#### 인증 시스템의 비침습적 통합

| 인증 단계 | 사용자 인지 수준 | 표현 방식 |
|-----------|------------------|-----------|
| 타이핑 패턴 수집 | 무의식 (알릴 필요 없음) | 없음 -- 백그라운드 진행 |
| 수집 상태 표시 | 미약 (궁금할 때만) | Inspector 하단에 작은 "Recording" 도트 |
| 인증 준비 완료 | 중간 (완성 시점에 알림) | Inspector 상단에 "Ready to certify" 배지 |
| 인증서 발행 | 명시적 (사용자 주도) | Publish 버튼 클릭으로 모달 진입 |

#### AI 어시스트의 비침습적 통합

| AI 기능 | 표현 시점 | 표현 강도 | 표현 위치 |
|---------|-----------|-----------|-----------|
| 맞춤법 | 타이핑 멈춤 후 2초 | 매우 약함 (wavy underline, opacity 0.3) | 인라인 |
| 문법 | 단락 이탈 후 | 약함 (dotted underline) | 인라인 |
| 스타일 | 전체 분석 후 | 매우 약함 (dashed underline, opacity 0.2) | 인라인 |
| 팩트 체크 | 전체 분석 후 | 중간 (Inspector 내) | Inspector |
| 요약 | 전체 분석 후 | 중간 (Inspector 하단) | Inspector |
| 글 구조 제안 | 요청 시에만 | 강함 (별도 패널) | 모달/시트 |

## 2. Zen 에디터 UX 심화

### 2.1 집중 도구 설계

#### Focus Mode (집중 모드)

기존 프로토타입의 Focus Mode 사양을 유지하면서, 세 가지 하위 변형을 제공한다.

```
Focus Mode 변형:

[Soft Focus]
  - 비활성 단락: opacity 0.4
  - 인라인 피드백: 숨김
  - Inspector: 숨김
  - 여백: 기본 유지
  - 적합: 전체 흐름을 보면서 현재 단락에 집중

[Deep Focus]
  - 비활성 단락: opacity 0.2 + blur(0.5px)
  - 인라인 피드백: 숨김
  - Inspector: 접근 불가
  - 여백: 상하 160px으로 확대
  - 적합: 한 단락에 완전 몰입

[Zen Focus] (최대 집중)
  - 비활성 단락: opacity 0.1 + blur(1px)
  - 모든 UI 요소 완전 숨김
  - 화면에 현재 단락 + 커서만 존재
  - 에디터 배경에 미세한 종이 텍스처 (선택적)
  - 전환: 화면 가장자리에 마우스를 가져가면 UI 복원 힌트 표시
  - 적합: 극한의 몰입, 장문 작성
```

#### Typewriter Mode (타자기 모드)

```
동작 원리:
  - 현재 입력 줄이 항상 화면 세로 중앙(또는 중앙에서 약간 위)에 위치
  - 새 줄이 추가될 때 기존 텍스트가 위로 스크롤
  - 스크롤 애니메이션: 부드러운 ease 300ms
  - 시각적으로 "타자기 롤러"가 굴러가는 느낌

설정:
  - 활성화: 설정 > Appearance > Typewriter Mode 토글
  - 키보드: Cmd+Shift+T
  - 현재 줄 위치: 화면 40% 지점 (정중앙보다 약간 위 -- 시선 편의성)

Focus Mode와의 조합:
  - Focus Mode + Typewriter Mode 동시 활성화 가능
  - 이 경우 "활성 줄"만 선명하고, 위아래 텍스트가 점진적으로 흐려짐
  - 가장 몰입감 높은 조합
```

#### 단어 수 목표 (Writing Goal)

```
동작:
  - 설정 또는 에디터 내에서 목표 단어 수 설정 (예: 1000단어)
  - 에디터 최하단에 극도로 미세한 프로그레스 바 표시
    - 색상: var(--text-disabled), 높이: 2px
    - 진행도에 따라 좌에서 우로 채워짐
    - 100% 달성 시: 짧은 축하 마이크로인터랙션
      (프로그레스 바가 var(--accent-verified) 녹색으로 전환 후 1초 뒤 fade out)
  - Focus Mode에서는 프로그레스 바도 숨김

위치 대안:
  - 스크롤바 트랙 위에 오버레이 (스크롤바 자체가 진행도 표시)
  - Inspector 통계 영역의 Words 숫자 옆에 작은 링형 프로그레스
```

### 2.2 Inspector 패널 노출/숨김 전략

#### 현재 프로토타입의 문제점

1. 마우스를 오른쪽 가장자리로 이동해야 한다는 것이 발견 불가능(undiscoverable)
2. 닫힌 상태에서 Inspector의 존재 자체를 인지하기 어려움
3. 에디터 본문 클릭으로 닫히는 것이 실수 유발 가능

#### 개선된 Inspector 진입점 설계

```
항상 보이는 요소 (Inspector 닫힌 상태):
  ┌─────────────────────────────────────────────────────┐
  │                                                 [i] │ <- 미니 아이콘
  │                                                  |  │ <- 세로 점선 (2px, opacity 0.15)
  │          [에디터 본문]                            |  │
  │                                                  |  │
  │                                                  |  │
  └─────────────────────────────────────────────────────┘

  [i] 아이콘: 16x16px, opacity 0.3, 호버 시 opacity 0.8
  세로 점선: 화면 우측 끝에서 8px 안쪽, 높이 40px, 세로 중앙 정렬
  역할: "여기에 뭔가 있다"는 시각적 어포던스(affordance)

열기 동작 (우선순위순):
  1. [i] 아이콘 클릭
  2. Cmd+I / Ctrl+I
  3. 마우스 우측 가장자리 500ms 호버 (기존 유지)
  4. 세 손가락 좌 스와이프 (트랙패드)

닫기 동작:
  1. Cmd+I 재입력
  2. Escape (포커스가 Inspector에 있을 때)
  3. Inspector 외부 클릭 (에디터 본문 클릭)
  4. [x] 닫기 버튼 (Inspector 상단 우측)
  5. 세 손가락 우 스와이프 (트랙패드)
```

#### Inspector 상태 기억

```
규칙:
  - 마지막 Inspector 열림/닫힘 상태를 로컬에 저장
  - 다음 방문 시 이전 상태로 복원
  - 단, Focus Mode 진입 시에는 무조건 닫힘
  - Focus Mode 해제 시 Focus Mode 진입 전 상태로 복원
```

### 2.3 AI 어시스트 비침습적 표현 방식

#### 인라인 피드백 (본문 내)

```
표현 계층:

Level 0 - 순수 밑줄 (기본 상태)
  - 본문 텍스트 아래에 밑줄만 표시
  - 밑줄 위 마우스 호버: 없음 (모바일 고려)
  - 밑줄 클릭/탭: 인라인 팝오버 등장

Level 1 - 인라인 팝오버 (클릭 시)
  ┌──────────────────────────────────┐
  │  [Spelling]  "recieve" -> "receive"  │
  │                                      │
  │  [Fix]  [Ignore]  [Learn]            │
  └──────────────────────────────────┘
  - 본문 밑줄 클릭 시 해당 위치 바로 아래에 팝오버
  - 팝오버 폭: 최대 300px, 내용에 맞게 조절
  - 배경: var(--surface-primary) + shadow
  - 3개 액션 버튼: Fix(수정 적용), Ignore(무시), Learn(사전에 추가)

Level 2 - Inspector 연동 (전체 보기)
  - Inspector 내 리뷰 아이템과 본문 밑줄이 양방향 연결
  - Inspector에서 아이템 클릭 -> 본문 해당 위치로 스크롤 + 하이라이트
  - 본문 밑줄 클릭 -> Inspector에서 해당 아이템 하이라이트
```

#### 사이드 거터 (Gutter) 마커

```
위치: 에디터 본문 좌측 여백 (좌측 패딩 영역 내)

표현:
  - 피드백이 있는 줄의 왼쪽에 작은 컬러 도트 (4px)
  - 색상: 해당 피드백의 심각도 색상
  - opacity: 0.4 (기본) -> 0.8 (해당 줄에 커서 위치 시)

장점:
  - 본문 텍스트를 전혀 변형하지 않음
  - 한눈에 문서 전체의 피드백 밀도를 파악 가능
  - 스크롤바에 미니맵처럼 표시 가능 (선택적)

설정:
  - "인라인 밑줄 + 거터 도트" (기본)
  - "거터 도트만" (본문 깔끔하게)
  - "인라인 밑줄만" (전통적)
  - "없음" (Focus Mode에서 자동)
```

## 3. Human Written 인증 UX

### 3.1 타이핑 분석 진행 표시

#### 디자인 원칙: "수집하되, 감시하지 마라"

타이핑 분석은 사용자에게 안심과 신뢰를 줘야 한다. "감시당하는 느낌"이 아니라 "보호받는 느낌"을 전달해야 한다.

#### 표시 방식

```
상시 표시 (Inspector 내):
  ┌─ Inspector 하단부 ─────────────────────┐
  │                                        │
  │  [Recording 도트]  Writing session      │
  │                   active for 47 min     │
  │                                        │
  │  Keystroke patterns: 12,847 captured    │
  │  Confidence: ████████░░ 82%            │
  │                                        │
  └────────────────────────────────────────┘

  Recording 도트:
    - 녹색 원형 도트 (6px)
    - 부드러운 pulse 애니메이션 (opacity 0.6 -> 1, 2s 주기)
    - 수집 중단 시: 회색으로 변경, pulse 정지

  Confidence 바:
    - 10칸 블록 프로그레스
    - 70% 미만: 회색 (아직 데이터 부족)
    - 70-89%: 녹색 (인증 가능)
    - 90%+: 진한 녹색 (높은 신뢰도)
    - 100%에 도달하면 "Ready to certify" 배지 표시
```

#### Inspector 닫힌 상태에서의 표시

```
에디터 우하단에 미니 상태 표시:
  - 녹색 도트 (4px) + "Recording" 텍스트 (10px, opacity 0.3)
  - Focus Mode에서는 이것도 숨김
  - 호버 시 opacity 0.6으로 증가 + 툴팁: "타이핑 패턴을 기록 중입니다"
```

#### 프라이버시 안심 메시지

```
첫 세션 시작 시 (한 번만):
  ┌──────────────────────────────────────────────┐
  │  [Shield 아이콘]                              │
  │                                              │
  │  "HumanWrites는 타이핑 리듬(간격, 속도)만    │
  │   분석합니다. 입력 내용은 분석에 사용되지      │
  │   않으며, 모든 데이터는 기기에서 처리됩니다."  │
  │                                              │
  │              [확인]  [자세히 보기]             │
  └──────────────────────────────────────────────┘

  - 에디터 하단에 토스트 형태로 표시
  - 8초 후 자동 사라짐 또는 "확인" 클릭으로 즉시 닫기
  - "자세히 보기"는 설정 > Privacy 섹션으로 이동
```

### 3.3 인증서 발행 플로우 사용자 경험

#### 현재 프로토타입 개선

기존 2단계(미리보기 -> 발행)를 유지하되, 감정적 여정(emotional journey)을 강화한다.

```
감정 곡선:

  기대감 ─────── 긴장감 ─────── 안도감 ─────── 자부심
    |               |               |              |
  [미리보기]    [분석 중...]     [발급 완료]    [공유]
```

#### 상세 플로우

```
Phase 1: 진입 (기대감)
  트리거: Inspector의 "Publish" 버튼 또는 Cmd+P

  조건 확인:
    - 최소 단어 수 충족? (기본: 100단어)
    - 타이핑 분석 confidence 70% 이상?
    - 미충족 시: 부드러운 안내 메시지
      "인증에 필요한 타이핑 데이터가 아직 부족합니다.
       조금 더 작성한 후 시도해주세요. (현재 신뢰도: 58%)"

Phase 2: 미리보기 (긴장감)
  모달 등장:
    - 배경 블러 + 오버레이 fade in
    - 모달 scale(0.95) -> scale(1) 등장
    - 인증서 미리보기가 "봉인된 봉투에서 꺼내는" 듯한 느낌

  인증서 미리보기 카드:
    - 크림색 배경 (var(--surface-certificate))
    - 방패 배지 (badge-appear 애니메이션)
    - 글 제목 (Playfair Display Italic)
    - 저자명
    - 통계 (단어 수, 작성 시간, 편집 횟수)

  검증 항목:
    - 순차 등장 애니메이션 (각 150ms 간격)
    - [v] 타이핑 패턴 일관성 검증
    - [v] AI 도구 미사용 확인
    - [v] 문체 지문(stylometric fingerprint) 분석
    - 각 항목 옆에 [?] 아이콘: 호버 시 검증 방법 설명 툴팁

Phase 3: 발급 (안도감)
  "인증서 발급" 버튼 클릭 후:
    - 버튼 텍스트 -> 스피너 교체 (200ms crossfade)
    - 검증 항목 아이콘이 체크에서 "분석 중" 스피너로 전환
    - 서버 통신 (1-3초)
    - 체크 아이콘이 순차적으로 녹색 체크로 변경
    - 전체 카드에 미세한 glow 효과 (box-shadow 녹색 번짐)
    - 버튼: 스피너 -> "발급 완료!" + 체크 아이콘 (배경 녹색)

Phase 4: 공유 (자부심)
  발급 완료 후 (500ms 후 자동 전환):
    - 모달 내용이 crossfade로 교체
    - 인증서 링크 표시 + 원클릭 복사
    - SNS 공유 버튼 (Twitter/X, LinkedIn, 이메일)
    - QR 코드 (인쇄/오프라인 용도)
    - "인증서 페이지 보기" 링크
    - 닫기 시: 에디터로 복귀, 문서에 인증 배지 자동 부착
```

## 4. AI 어시스트 UX

### 4.1 실시간 맞춤법/문법 피드백의 비침습적 표시

#### 기존 프로토타입의 3단계 모델 유지 + 개선

```
타이밍 규칙 (개선):

  [타이핑 중]
    - 피드백 갱신 완전 중단
    - 이유: 글쓰기 흐름을 절대 끊지 않음

  [타이핑 멈춤 + 1.5초]
    - 현재 단어의 맞춤법만 체크 (단어 단위)
    - 표시: 해당 단어 아래 미세한 밑줄 (opacity 0.2에서 시작, 500ms에 걸쳐 0.3으로)

  [타이핑 멈춤 + 3초]
    - 현재 단락 전체 분석
    - 문법, 문장 구조 피드백 추가

  [타이핑 멈춤 + 8초]
    - 전체 문서 분석 (백그라운드)
    - 스타일, 일관성, 가독성 분석
    - 결과는 Inspector에만 반영 (인라인 표시 안 함)

  [단락 이탈 (커서가 다른 단락으로)]
    - 이전 단락의 피드백 즉시 인라인 표시
    - 이유: 사용자가 해당 단락의 편집을 완료했다고 판단
```

#### 피드백 우선순위 표시

```
동일 위치에 여러 피드백이 겹칠 때:

  우선순위: 맞춤법 오류 > 팩트 체크 > 문법 > 스타일

  표시 규칙:
    - 가장 높은 우선순위의 밑줄만 표시
    - 팝오버에서는 모든 피드백을 목록으로 표시
    - Inspector에서는 심각도별 필터 제공
```

#### 인라인 팝오버 상세

```
맞춤법 팝오버:
  ┌──────────────────────────────────────┐
  │  Spelling                            │
  │  "recieve" -> "receive"              │
  │                                      │
  │  [Fix]  [Ignore]  [Add to Dictionary]│
  └──────────────────────────────────────┘
  - Fix: 원클릭 수정 (자동 적용, 팝오버 닫힘)
  - Ignore: 이 항목만 무시 (현재 세션)
  - Add to Dictionary: 사용자 사전에 추가 (영구)

문법 팝오버:
  ┌──────────────────────────────────────┐
  │  Grammar                             │
  │  주어-동사 일치 오류                  │
  │  "데이터들이 보여준다" ->              │
  │  "데이터가 보여준다"                   │
  │                                      │
  │  [Fix]  [Ignore]                     │
  └──────────────────────────────────────┘
```

### 4.2 팩트 체크 결과 표시

#### 표시 위치 전략

팩트 체크는 맞춤법과 달리 더 신중한 표시가 필요하다. 잘못된 팩트 체크 결과가 글쓰기를 방해할 수 있기 때문이다.

```
인라인 표시:
  - 팩트 체크 대상 문장/구절에 dotted underline (#991B1B, opacity 0.3)
  - 호버 시 팝오버:

  ┌──────────────────────────────────────────┐
  │  [!] Fact Check                          │
  │                                          │
  │  "Earth's population reached 9 billion   │
  │   in 2024"                               │
  │                                          │
  │  UN 데이터에 따르면 2024년 세계 인구는     │
  │  약 81억 명입니다.                        │
  │                                          │
  │  Sources:                                │
  │  - UN Population Division (2024)         │
  │  - World Bank Data                       │
  │                                          │
  │  [Review]  [Ignore]  [Mark as Correct]   │
  └──────────────────────────────────────────┘

  Review: Inspector의 해당 항목으로 포커스 이동
  Ignore: 이 팩트 체크 결과 무시
  Mark as Correct: "내가 맞다" - AI 결과 무시 + 학습

Inspector 표시:
  - 리뷰 아이템으로 표시
  - 배지: "Fact Check" (적색)
  - 출처 링크 포함
  - 신뢰도 표시: "확인 필요" / "높은 확률로 오류" / "출처 불명"
```

#### 팩트 체크 신뢰도 표시

```
레벨 1 - 정보 제공 (회색)
  "이 수치의 최신 데이터를 확인해보세요"
  -> 오류가 아닐 수 있지만 검증 권장

레벨 2 - 주의 (황색)
  "이 정보는 출처에 따라 다를 수 있습니다"
  -> 여러 출처에서 상충하는 정보 발견

레벨 3 - 경고 (적색)
  "이 사실 관계에 오류가 있을 가능성이 높습니다"
  -> 신뢰할 수 있는 출처와 명확히 불일치
```

### 7.1 WCAG 2.1 AA 준수 방안

### 7.3 RTL 언어 지원 고려

#### 현재 우선순위: 낮음 (Phase 5 이후)

한국어와 영어가 1순위이므로 RTL은 향후 고려사항이지만, 기초 설계를 미리 해둔다.

#### RTL 대응 설계 원칙

```
CSS 논리 속성 사용:
  - margin-left -> margin-inline-start
  - padding-right -> padding-inline-end
  - text-align: left -> text-align: start
  - left: 0 -> inset-inline-start: 0

레이아웃 미러링:
  - 에디터 본문: 우에서 좌로 텍스트 방향
  - Inspector: 좌측에 배치 (기존 우측에서 반전)
  - 설정 네비게이션: 우측에 배치
  - 아이콘: 방향성 있는 아이콘 미러링 (화살표 등)

폰트:
  - 아랍어: 'Noto Naskh Arabic'
  - 히브리어: 'Noto Serif Hebrew'
  - 숫자: 서양 숫자 유지 (통계, 날짜)

구현:
  <html dir="rtl" lang="ar">
  - CSS logical properties 전면 사용
  - Flexbox/Grid의 방향 자동 반전 활용
```

#### 양방향 텍스트 (BiDi) 처리

```
에디터 내 양방향 텍스트:
  - 단락 단위로 방향 자동 감지
  - 첫 강력 방향 문자(strong directional character)로 단락 방향 결정

```

3. 글쓰기 도구 + 인증 통합 (경쟁 없는 영역)
   - 별도 도구가 아닌 글쓰기 경험 자체에 인증이 내장
   - 검증 가능한 인증서 + 공유 가능한 검증 페이지
