## 1. 시작하기

### 앱 접속

브라우저 주소창에 `humanwrites.app`을 입력하면 HumanWrites 홈 화면으로 이동합니다. HumanWrites는 **데스크탑 전용** 서비스로, 모바일과 태블릿 환경은 현재 지원하지 않습니다. 화면 너비 1440px 이상의 환경을 권장합니다.

### Google 계정으로 로그인

1. 홈 화면에서 "Google로 계속하기" 버튼을 클릭합니다.
2. Google 계정 선택 화면이 나타나면 사용할 계정을 선택합니다.
3. HumanWrites의 권한 요청 화면에서 "허용"을 클릭합니다.
4. 인증이 완료되면 에디터 화면으로 자동 이동합니다.

로그인 후 세션은 안전하게 유지됩니다. 로그아웃은 우측 상단 프로필 영역에서 할 수 있습니다.

> **참고**: 현재 MVP 버전은 Google 계정 로그인만 지원합니다. GitHub, Apple 계정 연동은 추후 업데이트에서 제공될 예정입니다.

## 2. 글 쓰기 — Zen 에디터

### 에디터 화면 구조

로그인 후 나타나는 화면이 Zen 에디터입니다. 글에만 집중할 수 있도록 불필요한 UI 요소를 최소화한 미니멀 인터페이스로 설계되어 있습니다. 화면 대부분은 글쓰기 영역이며, 상단에 얇은 도구모음, 우측에 Inspector 패널 토글 버튼이 있습니다.

### 제목 입력

에디터 최상단의 큰 텍스트 입력 영역에 글 제목을 입력합니다. 제목 영역을 클릭하거나 페이지 로드 시 자동으로 포커스됩니다.

### 본문 입력

제목 아래 본문 영역을 클릭하면 커서가 이동합니다. 일반 텍스트 입력과 동일하게 타이핑하면 됩니다. 한국어 IME(입력기)를 완전히 지원하므로 한글 조합 중에도 자연스럽게 입력됩니다.

### 단락 포커스

현재 커서가 위치한 단락은 진하게 표시되고, 다른 단락은 약간 흐려집니다. 글의 한 부분에 집중할 수 있도록 돕는 기능입니다. 다른 단락을 클릭하면 포커스가 이동합니다.

### Markdown 단축키

에디터에서 Markdown 문법을 직접 입력하면 즉시 서식이 적용됩니다.

| 입력 방법 | 결과 |
|-----------|------|
| `# 제목` + Space | 제목 1 (H1) |
| `## 제목` + Space | 제목 2 (H2) |
| `### 제목` + Space | 제목 3 (H3) |
| `**텍스트**` | **굵게** |
| `*텍스트*` | *기울임* |
| `- 항목` + Space | 글머리 기호 목록 |
| `1. 항목` + Space | 번호 목록 |
| `> 텍스트` | 인용문 |
| `---` + Enter | 구분선 |

### 자동 저장

작성 중인 글은 브라우저 내부 저장소(IndexedDB)에 자동으로 저장됩니다. 별도의 저장 버튼을 누르지 않아도 됩니다. 브라우저를 닫고 다시 열어도 마지막으로 작성한 내용이 그대로 유지됩니다.

## 3. 테마 변경 — Light / Dark 모드

화면 우측 상단의 테마 전환 버튼(해 또는 달 아이콘)을 클릭하면 Light 모드와 Dark 모드를 전환할 수 있습니다.

- **Light 모드**: 흰색 배경, 진한 본문 텍스트. 밝은 환경에서 가독성이 좋습니다.
- **Dark 모드**: 어두운 배경, 밝은 본문 텍스트. 어두운 환경이나 장시간 집필 시 눈의 피로를 줄여줍니다.

테마 설정은 브라우저에 저장되며, 다음 접속 시에도 동일한 테마가 유지됩니다. 두 모드 모두 WCAG AA 수준의 색상 대비(4.5:1 이상)를 충족하도록 설계되어 있습니다.

## 4. Inspector 패널

Inspector 패널은 글쓰기를 방해하지 않도록 기본적으로 숨겨져 있습니다. 오른쪽에서 슬라이드인 방식으로 열립니다.

### 패널 열기 / 닫기

- **열기**: 화면 우측 상단의 Inspector 버튼(막대 그래프 아이콘)을 클릭합니다.
- **닫기**: 패널 우측 상단의 X 버튼을 클릭하거나, 키보드 단축키를 사용합니다.

### 탭 구성

Inspector 패널 상단에는 두 개의 탭이 있습니다.

#### 통계 탭 (Stats)

현재 문서의 기본 통계를 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 단어 수 | 본문의 총 단어 수 |
| 단락 수 | 본문의 총 단락 수 |
| 읽기 시간 | 평균 독서 속도 기준 예상 읽기 시간 (분) |
| 문자 수 | 공백 포함 총 문자 수 |

통계는 글을 입력하는 동안 실시간으로 업데이트됩니다.

#### 리뷰 탭 (Review)

AI가 감지한 맞춤법 및 문법 이슈 목록이 표시됩니다. 각 항목을 클릭하면 에디터에서 해당 위치로 이동합니다. 리뷰 탭 사용법은 [5장 AI 맞춤법 검사](#5-ai-맞춤법-검사)에서 자세히 설명합니다.

## 5. AI 맞춤법 검사

HumanWrites의 AI 어시스트 기능은 **맞춤법과 문법만 확인**합니다. AI가 글의 내용을 대신 작성하거나 문장을 생성하지 않습니다. AI는 어디까지나 교정 조수 역할에 그칩니다.

### 인라인 밑줄 표시

AI가 문제를 감지하면 해당 단어나 구문 아래에 색깔 있는 밑줄이 나타납니다. 밑줄은 반투명하게 표시되어 글 읽기를 방해하지 않도록 설계되어 있습니다.

| 밑줄 종류 | 의미 |
|-----------|------|
| 갈색 물결 밑줄 | 맞춤법 오류 (spelling) |
| 남색 파선 밑줄 | 문법 오류 (grammar) |

### Inspector 리뷰 탭에서 확인

Inspector 패널의 리뷰 탭을 열면 감지된 모든 이슈 목록을 한눈에 볼 수 있습니다. 각 항목 카드에는 다음 정보가 표시됩니다.

- **유형 배지**: "맞춤법" 또는 "문법"
- **심각도**: 오류 / 경고 / 정보
- **설명**: 문제가 된 내용과 이유
- **제안**: AI가 권장하는 수정 표현 (있는 경우)

### 수정 적용

항목 카드에서 제안된 수정 표현이 있는 경우 "수정" 버튼이 나타납니다.

- "수정" 버튼 클릭 시: AI가 제안한 표현으로 해당 부분이 즉시 교체됩니다.
- "무시" 버튼 클릭 시: 해당 항목이 목록에서 제거되고, 에디터의 밑줄도 사라집니다.

항목 카드를 클릭하면 에디터에서 해당 위치로 커서가 이동합니다. 직접 수정하고 싶을 때 활용할 수 있습니다.

### 검사 실행 조건

AI 맞춤법 검사는 일정량의 텍스트가 입력된 후 자동으로 실행됩니다. 검사가 진행 중일 때 리뷰 탭에는 로딩 표시가 나타납니다. 검사 결과가 없으면 "이슈 없음" 메시지가 표시됩니다.

> **참고**: 팩트 체크, 스타일 제안, AI 문장 생성 기능은 현재 버전에서 제공하지 않습니다.

## 6. 인증서 발행

글쓰기가 완료되면 Human Written 인증서를 발행할 수 있습니다. 인증서는 타이핑 패턴 분석을 통해 해당 글이 사람이 직접 타이핑하여 작성했음을 증명하는 디지털 문서입니다.

### 인증서 발행 시작

에디터 상단 도구모음에서 "인증서 발행" 버튼을 클릭합니다. 인증 모달이 화면 중앙에 열립니다.

### 분석 과정 (4단계)

인증 과정은 다음 네 단계로 진행됩니다.

**1단계 — 분석 중 (Analyzing)**

글쓰기 세션 동안 수집된 타이핑 패턴 데이터를 분석합니다. 이 단계에서 평가하는 항목은 다음과 같습니다.

- **타이핑 속도 변동성**: 일정하지 않고 자연스럽게 변하는 타이핑 속도 패턴
- **오류 수정률**: 오타를 인지하고 수정하는 행동 패턴
- **일시정지 패턴 엔트로피**: 생각하면서 멈추거나 속도가 달라지는 인간 고유의 불규칙성

**2단계 — 분석 결과 확인 (Review)**

분석이 완료되면 결과 화면이 나타납니다.

- **종합 점수**: 0~100점 사이의 신뢰도 점수
- **타이핑 변동성**, **오류 수정률**, **패턴 엔트로피** 세부 수치
- **판정**: Certified 또는 Not Certified

판정이 **Certified**인 경우 "인증서 발행" 버튼이 활성화됩니다. **Not Certified**인 경우 인증서를 발행할 수 없으며, 더 자연스러운 타이핑 패턴이 감지되도록 글쓰기를 계속할 것을 권장합니다.

**3단계 — 서명 중 (Signing)**

"인증서 발행" 버튼을 클릭하면 Ed25519 비대칭 서명 알고리즘으로 인증서에 디지털 서명이 적용됩니다. 이 서명은 인증서 내용이 발행 후 변조되지 않았음을 오프라인에서도 검증할 수 있게 합니다.

**4단계 — 발행 완료 (Complete)**

인증서가 성공적으로 발행되면 격식체 스타일의 인증서 카드가 표시됩니다.

- 문서 제목, 작성자 이름
- 단어 수, 종합 점수
- "Certified" 인증 마크

### 인증서의 한계 (투명성 공지)

현재 버전(MVP)의 인증서는 **Layer 1** 기술만 사용합니다. 즉, 키스트로크 다이나믹스(타이핑 패턴)를 기반으로 판정합니다. 다음 시나리오는 현재 버전에서 탐지할 수 없습니다.

- AI가 생성한 텍스트를 읽고 그대로 타이핑하는 경우

이 한계는 인증서 자체에도 명시됩니다. 편집 패턴 분석(Layer 2)과 콘텐츠 통계 분석(Layer 3)은 이후 버전에서 도입됩니다.

## 7. 인증서 공유 및 검증

### 공유 버튼

인증서 발행이 완료되면 세 가지 공유 버튼이 나타납니다.

| 버튼 | 동작 |
|------|------|
| Share on X | X(구 Twitter)에 인증 내용과 검증 URL을 포함한 트윗 창이 열립니다. |
| LinkedIn | LinkedIn 공유 창이 열립니다. |
| Copy Link | 검증 URL이 클립보드에 복사됩니다. |

### 검증 URL

각 인증서에는 고유한 검증 URL이 부여됩니다.

```
humanwrites.app/verify/{shortHash}
```

이 URL을 통해 누구든지 해당 인증서의 진위를 확인할 수 있습니다. 검증 페이지에서 확인할 수 있는 정보는 다음과 같습니다.

- 문서 제목, 작성자, 단어 수
- 발행 일시
- 종합 점수 및 등급
- Ed25519 디지털 서명 값
- 공개키 URL (서명 독립 검증용)

### 오프라인 서명 검증

기술적으로 고급 사용자는 HumanWrites 서버에 접속하지 않고도 인증서 서명을 직접 검증할 수 있습니다.

1. `humanwrites.app/.well-known/humanwrites-public-key.pem`에서 공개키를 내려받습니다.
2. 인증서의 `signature` 값과 문서 해시(SHA-256)를 Ed25519 공개키로 검증합니다.

## 9. 자주 묻는 질문 (FAQ)

### Q. 모바일이나 태블릿에서 사용할 수 있나요?

현재 버전은 데스크탑 전용입니다. 모바일 및 태블릿 지원은 MVP 이후 이터레이션에서 제공될 예정입니다.

### Q. 작성 중인 글이 서버에 저장되나요?

현재 MVP 버전에서 글 본문은 브라우저 내부 저장소(IndexedDB)에만 저장됩니다. 클라우드 동기화 기능은 추후 버전에서 제공됩니다. 브라우저 데이터를 초기화하면 저장된 글이 삭제될 수 있으니 중요한 내용은 별도로 백업해 두시기 바랍니다.

### Q. 타이핑 내용(키 입력값)이 서버로 전송되나요?

전송되지 않습니다. 어떤 키를 눌렀는지(실제 키값)는 절대 서버에 전송하지 않습니다. 인증에 사용되는 데이터는 타이핑 속도, 키 누름 시간, 키 간격 등의 **통계 메트릭**뿐입니다. 개인정보 보호 설계 원칙에 따라 원시 키스트로크 데이터는 기기 밖으로 나가지 않습니다.

### Q. Not Certified 판정을 받았습니다. 어떻게 해야 하나요?

타이핑 패턴이 충분히 쌓이지 않았거나, 분석 알고리즘이 자연스러운 인간 타이핑 패턴으로 판단하기 어려운 경우 Not Certified가 나타날 수 있습니다. 글을 더 작성하거나, 자연스럽게 타이핑하면서 평소처럼 오타를 수정하는 과정도 그대로 유지하면 도움이 됩니다. 복사-붙여넣기보다 직접 타이핑하는 것이 인증 점수에 유리합니다.

### Q. 인증서의 유효기간이 있나요?

발행된 인증서는 기본적으로 만료 기간 없이 유지됩니다. 다만, 인증 알고리즘이 업데이트되면 이전 인증서는 당시 버전 기준으로 유지됩니다. 검증 페이지에서 인증서 발행 시점의 프로토콜 버전을 확인할 수 있습니다.

### Q. AI가 글을 대신 써주나요?

HumanWrites의 AI는 글을 대신 작성하지 않습니다. AI의 역할은 맞춤법과 문법 오류를 알려주는 것에 한정됩니다. 글의 내용과 문장은 전적으로 작성자 본인이 만들어야 합니다. 이것이 HumanWrites가 추구하는 핵심 가치입니다.

### Q. AI 맞춤법 검사가 작동하지 않는 것 같습니다.

다음 사항을 확인해 보세요.

- 본문에 충분한 텍스트(한 단락 이상)가 입력되어 있는지 확인합니다.
- Inspector 패널의 리뷰 탭을 열어 "검사 중..." 로딩 상태가 표시되는지 확인합니다.
- 이슈가 발견되지 않으면 "이슈 없음" 메시지가 표시됩니다. 맞춤법 오류가 없는 경우 정상입니다.
- 위 사항이 모두 정상임에도 작동하지 않는다면 페이지를 새로고침해 보세요.

### Q. 인증서 검증 URL을 공유했는데 상대방이 열 수 없다고 합니다.

검증 페이지(`humanwrites.app/verify/{shortHash}`)는 로그인 없이 누구든 접근할 수 있는 공개 페이지입니다. 주소가 정확히 복사되었는지 확인하세요. "Copy Link" 버튼을 사용하면 정확한 URL이 클립보드에 복사됩니다.

### Q. 계정이나 데이터를 삭제하고 싶습니다.

설정 화면에서 계정 삭제를 신청할 수 있습니다. 삭제 시 모든 문서 데이터와 인증 기록이 영구 삭제됩니다. GDPR 규정에 따라 데이터 내보내기(`데이터 다운로드`) 기능도 설정 화면에서 제공됩니다.
